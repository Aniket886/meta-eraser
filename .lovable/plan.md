

# Redesign PDF Audit Report + PWA-Aware AI Insights

## Summary

Rebuild the PDF report generator to produce a clean, professional, branded document with clear before/after comparison. When the user is on the website (browser), fetch AI insights via Groq before generating. When running as an installed PWA, skip AI entirely and generate a local-only anonymous report.

## What Changes

### 1. Add PWA detection utility
Create `src/lib/pwa-utils.ts` with a helper that checks `window.matchMedia('(display-mode: standalone)')` to detect if the app is running as an installed PWA.

### 2. Redesign `src/lib/pdf-report.ts` — Professional branded PDF
Complete rewrite of the PDF layout:

- **Cover page**: MetaClean logo/brand name, report title, date, user info (or "Anonymous" for PWA), file count summary
- **Executive Summary page**: Total fields removed vs kept, pie-chart-style visual stats, overall risk score
- **AI Insights section** (website only): Groq-powered privacy analysis with styled callout box
- **Per-file detail pages** with clear **Before → After** comparison:
  - File info header (name, type, size)
  - Two-column table: Field Name | Before Value | After Value | Status (REMOVED/KEPT)
  - Color-coded rows: red highlight for removed, green for kept
  - Summary bar per file showing removed/kept counts
- **Footer on every page**: "Generated by MetaClean • [date] • Confidential" + page numbers
- **Header on every page**: MetaClean brand bar with indigo accent

### 3. Update `src/lib/audit.ts` — Store after values
Add `afterValue` field to `AuditField` interface so the PDF can show before/after comparison. Update `generateAuditReport` to capture the after value from `metadataAfter`.

### 4. Update `src/pages/Dashboard.tsx` — PWA-aware PDF download
- Import the PWA detection utility
- In `handleDownloadPdfReport` and `handleDownloadAllPdfReports`:
  - If **PWA mode**: skip AI insights call, pass no user info (anonymous), generate PDF locally
  - If **browser mode**: fetch AI insights via Groq as currently done, include user info
- Pass `isPWA` flag to `generatePdfReport` so it can adjust branding text (e.g., hide "Powered by Groq" label)

### 5. Update edge function (no changes needed)
The existing `generate-report-insights` edge function stays as-is — it's simply not called in PWA mode.

## Technical Details

**PWA detection:**
```typescript
export function isPWA(): boolean {
  return window.matchMedia('(display-mode: standalone)').matches
    || (navigator as any).standalone === true;
}
```

**AuditField update:**
```typescript
export interface AuditField {
  name: string;
  originalValue: string;
  afterValue: string;      // NEW
  status: "removed" | "kept";
  reason?: string;
}
```

**PDF report options update:**
```typescript
interface ReportOptions {
  userName?: string;
  userEmail?: string;
  aiInsights?: string;
  isPWA?: boolean;  // NEW — controls anonymous mode
}
```
